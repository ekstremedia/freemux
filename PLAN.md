1. **Source**
   Improve the source workflow so it focuses on input media and probe details only.
   Support adding multiple files or a folder, show the selected media in a removable list, and keep the rest of the app aligned with multi-file behavior.
   Provide an efficient video preview with standard playback controls.

2. **Conversion**
   Rename the current Convert tab to a clearer label such as Settings or Conversion.
   Move conversion progress and command preview out of this tab and keep the profile workflow focused on editing settings.
   Fix the "Save as new" profile flow and simplify profile management so it feels deliberate and easy to use.

3. **Output**
   Use the Output tab for output paths, conversion progress, and command preview.
   Show a concise summary of the selected input files and the active conversion settings.
   Let the user set a single output filename for one file or a destination folder and per-file names for batch jobs.
   Make the command preview easy to copy, ensure paths are quoted when needed, and format the command for readability.

4. **Actions**
   Make the primary action a prominent Start Conversion button with an adjacent Cancel action while a job is running.
   Show both per-file progress and overall batch progress during conversion.
   Provide quick actions to open the output folder or open completed files directly after conversion.

The interface should remain compact, responsive, and desktop-like, with clear feedback and consistent controls across the workflow.
